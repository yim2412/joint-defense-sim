# v16.4 분산해양작전(DMO) — 설계

## 개념
미 해군 분산해양작전(Distributed Maritime Operations) 교리: 전력을 광역 분산해
적의 집중 포화 표적화를 회피하되, 분산의 대가로 상호 협동방어(CEC 엄호)가 약화된다.
**양날 트레이드오프**로 모델링 — 분산이 일방적 이득이 아니라 전술적 선택지가 되게 한다.

토글 `enable_dmo` (실험적, 기본 OFF). 어느 편대에서나 켤 수 있어 미 원형(대형함 광역분산)·
한국형(소형함 다수 분산) 둘 다 표현. 함정 수가 많을수록(한국 FFX 다수 편대) 분산 효과가 큼.

## 트레이드오프 두 축

### 이득 — 적 포화 분산 (구현 필요)
현재 `_pick_target`(engine_combat:1869)은 대함미사일 표적을 **max_hp 가중 랜덤**(위치 무관)으로
고른다 → 함정을 아무리 분산해도 적 포화가 그대로 집중된다. 이 갭을 메운다.
- DMO ON 시 표적선택을 **위협 접근축/위치 기반**으로 전환: 위협 위치에서 가까운(접근축 상) 함정에
  가중 → 광역 분산 시 멀리 떨어진(접근축 밖) 함정은 표적 확률이 급감 = 집중 포화 회피.
- 어뢰(기함 우선)는 현행 유지(잠수함은 기함을 노리는 게 현실적).

### 대가 — CEC 협동방어 약화 (기존 로직 자동 발현, 코드 최소)
`_friendly_defense`(engine_combat:3088~)는 이미:
- 각 함정의 자체 탐지거리 `own_detect_m` 초과 시 → CEC ON이면 중계 교전(Pk 패널티), OFF면 교전 불가
- `_select_defense_wpn`이 거리로 무기 선택 → SAM 사거리 밖이면 엄호 불가
→ **분산 반경만 키우면(`dmo_spread_km`) 원거리 함정 상호 엄호가 자동으로 약화**된다. 별도 코드 불요.

## 수정 파일
1. **engine_combat.py**
   - 함정 배치부(~1492): `enable_dmo` ON 시 분산 반경을 `dmo_spread_km`(기본 40km)로 확대
     (기존 `_FORMATION_RADIUS` 3~5km → DMO 시 광역). 결정론 보존(현 배치는 각도 결정적, 추가 random 금지).
   - `_pick_target`: DMO ON 분기 — 위협 위치 인자 받아 접근축 거리 기반 가중. OFF면 기존 경로 bit-identical.
2. **app_main.py**
   - 체크박스 "분산해양작전 DMO (실험적)" + 툴팁(기본 OFF — 기존 결과와 동일).
   - cfg 빌드(`isChecked()`) + 로드(`hasattr`) 3종세트. `dmo_spread_km`는 cfg 기본값으로(슬라이더는 후속).
3. **engine_core.py** — 변경 없음(별도 분산 편대 프리셋은 선택, 후속 판단).

## 하위호환 / 회귀
- 기본 OFF → 회귀 8×26 bit-identical 필수. `_pick_target` OFF 경로 순서·RNG 소비 불변.
- `enable_dmo`·`dmo_spread_km` 플래그명 고정(저장 시나리오 호환).

## 검증 (승격 게이트: 전술 옵션 유형)
- 회귀 PASS(OFF bit-identical)
- ON/OFF MC 기준값 측정 → 분산이 **의미있는 트레이드오프**인지(요격률·아군피격·생존함정 수).
  기대: 분산 ON 시 적 포화가 흩어져 일부 함정 생존↑, 그러나 CEC 엄호 약화로 개별 피격↑.
  역효과·버그(분산이 무조건 이득/손해) 없는지 확인.
- 코드 감사 `/code-review medium`.
- 기준값 메모리 `project-baseline-dmo`로 보존.

## 미결(구현 중 확정)
- 위치 기반 표적선택의 정확한 가중식(접근축 거리 역가중 vs 탐지반경 내 후보 필터). 구현 시 측정으로 튜닝.
- `dmo_spread_km` 기본값(40km)은 측정 후 조정 — 너무 크면 CEC 완전 붕괴, 너무 작으면 효과 미미.
