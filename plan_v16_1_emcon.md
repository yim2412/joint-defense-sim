# v16.1 — 전자전·능동 방사 역탐지 (EMCON 딜레마)

> 정본 설계 문서. 난이도 **높음**(엔진 부모 메서드 변경 포함) → Opus/code-review high.
> 핵심 한 줄: **방사(레이더·능동소나) = 탐지 우위 ↔ 위치 노출(ARM·어뢰 반격)**. 현대 해전의 EMCON(Emission Control) 딜레마를 엔진에 인과로 심는다.

## 1. 현황 — 이미 있는 것 (재사용)

| 메커니즘 | 위치 | 동작 |
|----------|------|------|
| ARM 미사일 | `engine.py` ENEMY_DB 3종(Kh-31P·LD-10·Kh-58U, `is_arm=True`) | 적 항공기가 보유 시 발사. `대방사미사일` 타입, ECM 무효(역추적) |
| ARM 회피 | `engine_v7._arm_radar_off_check`(2732) | 아군이 ARM 120km 접근 탐지 시 레이더 OFF(`radar_off_until`) → `_check_hits`(3996)에서 ARM 빗나감 |
| 레이더 OFF 상태 | `FriendlyShipObj.radar_off_until`(1082) | ARM 회피 전술 + RL 레이더 레버(v15.11.01 `_apply_tactical_choice`) |
| 능동 소나 | `_sonar_eq_pd`(2050) | `for active in (False, True)` — **항상 수동+능동 max**. 능동이 탐지거리 김(왕복 TL·TS·잔향) |
| 적 잠수함 어뢰 | `_enemy_fire`(2554, **부모 메서드**) + `_pick_target(is_torpedo)`(1815) | 적 잠수함이 어뢰 발사, 기함 우선. 음향 기만기·회피 기동으로 방어 |

## 2. 갭 — v16.1이 메울 것

1. **ARM 발사/유도가 아군 레이더 방사 상태와 무관** — 적이 ARM을 그냥 쏜다. "아군이 레이더를 켜서 신호를 흘렸기 때문에 적 ESM이 포착→ARM 유도"라는 인과가 없음. 아군 레이더 OFF는 ARM '회피'만 할 뿐, 적의 ARM '발사 결심'을 막지 못함.
2. **능동 핑이 역탐지 비용 없이 공짜** — 능동 소나는 항상 켜진 셈(탐지 우위만 받고, 적에게 내 위치 노출되는 페널티 없음). EMCON 선택지가 아님.

## 3. 두 seq로 분할

### v16.01.01 — ESM→ARM 역탐지 (레이더 방사 딜레마)
- **끝상태**: 아군 함정이 레이더 ON(방사 중)일 때만 적 ESM이 신호 포착 → 그 함정 방향으로 ARM 발사·유도 유효. 레이더 OFF면 적 ESM 미포착 → ARM 발사 보류(또는 마지막 알려진 위치로 발사해 명중 급감).
- **구현 위치**: `_enemy_fire`의 ARM 무기 분기. ARM 발사 결심 전에 표적 함정 `radar_off_until` 체크.
  - 레이더 ON → 정상 발사.
  - 레이더 OFF → 발사 보류(쿨다운 후 재평가) 또는 stale 위치 발사(`_check_hits`에서 추가 Pk 페널티).
- **부모 무수정 전략**: `_enemy_fire`는 부모 TimeStepEngine 메서드(단발+전장 공통). ARM 인과는 두 모드 다 적용돼야 현실적 → **부모에 얹되 `enable_esm_arm` 플래그로 게이트**(기본 OFF면 기존 동작 bit-identical, 회귀 보존). 훅 추출이 어려우면 플래그 분기 1곳으로 최소 침습.
- **트레이드오프**: 레이더 OFF = ARM 봉쇄 ↔ 대공 탐지·교전 저하(이미 `radar_off_until`가 탐지 막음). 진짜 EMCON 선택.
- **🔑 RL 연계 보너스**: v15.11.01 레이더 ON/OFF 레버가 "현 균형풀 ARM 약해 OFF 열등"이라 RL이 안 쓰는 레버였음(메모리). **v16.01.01이 ESM/ARM을 연동하면 그 레버가 비로소 진짜 트레이드오프를 획득** → RL 행동공간이 의미있어짐. v16이 RL을 다시 살리는 구체 사례.

### v16.01.03 — 능동 소나 핑 역탐지 (어뢰 반격 딜레마) ※번호: ESM가 16.01.01, ARM 실효화가 16.01.02 차지 → 능동소나는 16.01.03
- **현황(조사 완료)**: `_sonar_eq_pd`(2062)가 `for active in (False, True)` max로 **능동을 항상 공짜 사용**. `_asw_detect_check`(3546)가 아군 항공기 능동(디핑/소노부이)으로 잠수함 탐지→어뢰 투하. 적 잠수함은 `hidden_until` 은닉 후 `_enemy_fire`(2614 '어뢰' in m_name)로 어뢰 발사.
- **갭**: 능동 핑 사용 시 적 잠수함이 역탐지하는 인과 없음(능동이 비용 0).
- **끝상태**: 능동 핑으로 적 잠수함 탐지 성공 시 → 적 잠수함이 아군 위치 역포착 → ①어뢰 반격 촉진(은닉 해제 + 발사 앞당김) + ②회피 기동(접촉 급감). 수동 운용은 은밀(역탐지 없음)하나 탐지 약화.
- **부모 무수정 전략**: `enable_sonar_emcon` 플래그 **기본 OFF로 현행 bit-identical 보존**(현행=능동 max·역탐지 없음). ON 시에만 능동 탐지 성공→역탐지. 역탐지 결과의 잠수함 거동 변경은 BattleEngine 전용 상태/훅으로(부모 _sonar_eq_pd·_enemy_fire는 플래그 게이트만).
- **🔑 ARM 교훈 적용**: 메커니즘만으론 부족 — 적 잠수함이 교전에 등장·접근해야 효과(대잠 복합·잠수함 복합 포화 시나리오로 측정). ON/OFF 효과를 실제 측정해 어뢰 반격↑·교환비 변화 확인 후에야 검증 완료.
- **열린 결정(합의 필요)**: A. 역탐지 효과 = 어뢰반격 / 회피 / 둘다. B. 능동/수동을 사용자 토글로 명시 선택할지(C결정) vs EMCON ON 시 능동 고정+역탐지(수동은 후속 RL 소나 레버).

## 4. 공통 설계 원칙

- **부모 무수정 우선**: 훅 추출/BattleEngine 한정. ARM·소나는 단발도 영향이라 부모에 얹는 게 불가피 → **`enable_xxx` 플래그 기본 OFF로 회귀 bit-identical 보존**이 안전망.
- **`enable_xxx` 3종세트**(체크박스+isChecked+hasattr), 기본 OFF, (실험적) 레이블. 플래그명: `enable_esm_arm`, `enable_active_sonar_counter`(확정 시 변경 금지).
- **각 seq**: 회귀 PASS(플래그 OFF) → ON/OFF 기준값 측정 → code-review high → 신기능 체크리스트 8항목 → spec_db/changelog/_PLANS/빌드.
- **환경물리 성격**: EMCON은 "항상 존재하는 환경물리"에 가까우나, ARM/역탐지는 전술 선택 성격도 큼 → 검증 후 승격 시 기본값 결정([[feedback_default_on_policy]] vs 선택 옵션). 도입 단계는 기본 OFF.

## 5. 진행 순서
v16.01.01(ESM→ARM) 먼저 — 레이더 레버 재활성 보너스가 크고, ARM 인프라가 더 성숙(is_arm·radar_off 완비). 그 후 v16.01.02(능동 소나 역탐지).

## 6. 열린 결정 — ✅ 합의 완료 (2026-06-24)
- **A. ARM 레이더 OFF 시 거동 → ②stale 위치 발사 + 명중 급감**. 적이 마지막 포착 위치로 ARM을 쏘되 레이더 OFF라 명중률 급감(완전 봉쇄는 비현실 — 적도 바보가 아님). 마지막 레이더 ON 시점 함정 위치를 추적(`FriendlyShipObj.last_emit_pos`류)해 ARM이 그 고정 좌표로 유도.
- **B. 적용 범위 → 둘 다 단발+전장 공통**. ESM/ARM·소나 역탐지 모두 부모 메서드에 얹되 `enable_xxx` 플래그 **기본 OFF로 회귀 bit-identical 보존**이 안전망. 회귀 골든은 OFF 경로라 유지.
- **C. 능동 소나 EMCON → 능동/수동 선택 토글**. 현재 "항상 능동"을 토글로 제어. 능동 ON=탐지 우위+역탐지 위험, 수동=은밀+탐지 약화. EMCON 딜레마를 선택지로 명확히 표현.
