# 로드맵 상세 검토 — v11~v20

> **감사 체크포인트 정책** (정본: CLAUDE.md '감사 정책')
> 마이너 버전마다 빌드 직전, 변경 유형에 맞는 감사 1회:
> 엔진/로직→`/code-review`, DB/수치→현실성 수치감사, UI→회귀확인, 아키텍처전환→코드감사+로직트레이스.
> 메이저 블록(v11·v12…) 완료 시 → 전체 회귀 MC + 누적 수치감사.

---


각 항목을 **3차원**으로 검토한 현실성 분석. `_PLANS`(정본)의 난이도·범위는 이 문서 결론에 맞춰 보정.

- **구현**: 실제 만들 수 있나 — 공수(1인 기준), 기술 위험, MVP 분할
- **군사**: 교리·수치·무기 정확도 — 비현실적 가정 교정
- **범위**: 욕심/순서 — 핵심·축소·보류 판정

공수 단위: `일`=1~2일, `주`=5일 환산. 1인 개발 기준 순수 코딩+검증 시간.

## v11 — 기반 정비 (전술급 토대)

### v11.1 검증 시뮬 실행 — 난이도 **낮음**(유지)
- **구현**: 공수 ~2일. 기존 `monte_carlo_v7` 재사용, 위험 낮음. MVP 불필요.
- **군사**: baseline 시나리오의 **대표성**이 핵심. '기동전단 기본 vs 랴오닝 항모전단' 하나로는 부족 — 저위협(고속정 살보)·고위협(HGV 포화) 양극단 2개 추가 권장.
- **범위**: 적정. 모든 후속 버전의 비교 기준이므로 **필수 선행**.

### v11.2 차기 계획 수립 — 난이도 **낮음**(유지)
- **구현**: 코딩 아님. 문서 작업.
- **군사**: 무관.
- **범위**: 이 문서로 사실상 대체됨. → **흡수/완료 처리 가능**.

### v11.3 설정 패널 UI 정리 — 난이도 **중간**(유지)
- **구현**: 공수 ~1주. QTabWidget 재배치는 단순하나 기존 시그널 연결(40+ 체크박스) 재배선 시 누락 위험. 회귀 테스트 필요.
- **군사**: 무관.
- **범위**: 적정. 단 v12~v16이 항목을 계속 추가하므로 **v16 직후로 미루는 게 효율적**(중복 재배치 방지). → 순서 재조정 권장.

### v11.4 MC 성능 최적화 — 난이도 **중간→높음** (상향)
- **구현**: "numpy 벡터화"는 현 객체지향 루프(`MissileObj.update`)를 배열 연산으로 전면 재작성 = 대규모 리팩토링. 공수 ~2~3주. **위험 큼**: 벡터화 후 기존 결과와 미세 불일치(부동소수점·난수 순서) → 회귀 검증 필수.
- **군사**: 무관(순수 성능).
- **범위**: "10,000회"는 욕심. **MVP 분할**:
  - MVP-A(2일): SimFrame을 MC 모드에서 비활성화 + 배치 크기 튜닝 → **2~3배** 즉시 확보.
  - MVP-B(2~3주): numpy 벡터화 → 추가 5~10배. v14.2/v17 선행으로 필요할 때만.

### v11.5 시나리오 라이브러리 — 난이도 **낮음**(유지)
- **구현**: 공수 ~2~3일. cfg JSON 4종 + 로더. 기존 save/load_scenario 재사용.
- **군사**: 시나리오 교리 정확도가 가치를 결정. 실제 적 편성·접근 방위 고증 필요(이미 ENEMY_FLEET_PRESETS 있음).
- **범위**: 적정. v11.1 baseline과 통합하면 일석이조.

## v12 — 물리 엔진 고도화

### v12.1 실제 유도 알고리즘 (PNG) — 난이도 **매우 높음**(유지)
- **구현**: 공수 ~3~4주. PNG 식 자체는 단순(`a = N·V_c·λ̇`)하나, **모든 교전 결과가 바뀜 → 전면 재밸런싱**이 진짜 비용. 위험 매우 큼.
- **군사**: PNG는 실제 유도 법칙으로 정확. 단 빠진 요소 — **항법상수 N(3~5), 미사일 최대 기동 g한계, 탐색기 시야각(FOV)**. 이게 없으면 "무한 기동 미사일"로 비현실. SAM은 ~30g, 대함미사일 종말 ~10g 한계 반영 필수.
- **범위**: 너무 큼. **MVP**: 전(全) 비행 PNG는 과함 → **종말 단계(<10km)만 PNG**, 중간 비행은 현 직선 모델 유지. 회피 기동의 핵심만 잡힘.

### v12.2 수중 음향 전파 모델 — 난이도 **매우 높음**(유지)
- **구현**: SONAR 방정식(`SL−2·TL+TS−(NL−DI) > DT`)은 구현 가능. TL 정밀 계산(Ray-tracing)은 무거움. 공수 ~3~4주.
- **군사**: **중대 교정** — 수렴지대(CZ)는 **심해(수심 >1000m)에서만** 발생. 동해(평균 1500m)는 가능하나 **서해(평균 44m)는 CZ 물리적으로 불가**. 현 _PLANS는 무조건 CZ 적용처럼 읽힘 → 해역 수심 분기 필수. 서해는 표면덕트·해저반사만.
- **범위**: 큼. **MVP**: 완전 Ray-tracing 대신 **수심대역별 TL 룩업테이블 + 기존 수온약층(v9.12) 강화**. CZ는 동해 한정 보너스.

### v12.3 함정 생존 능력 동적 모델링 — 난이도 **매우 높음**(유지)
- **구현**: 공수 ~3~4주. GZ곡선·격실배수는 조선공학 영역 — 정밀 모델은 과함. 위험: 데이터 부재(함별 격실 배치는 기밀).
- **군사**: 개념은 타당. 단 **실측 데이터 없음** → 추정 모델일 수밖에. "침몰 타임라인 예측"은 정밀 주장하면 거짓 정확도(false precision).
- **범위**: 욕심. **MVP**: GZ곡선·격실 단위 시뮬은 보류. **간이 모델** — 피격 위치(수선하/상) + 누적 침수 게이지(0~100%) → 임계 초과 시 점진적 speed/기능 저하 후 침몰. 현 정적 계수보다 현실적이면서 구현 1주.

### v12.4 동적 기상 변화 — 난이도 **중간**(유지)
- **구현**: 공수 ~3~4일. `wx`를 시간함수로 전환은 단순. 위험 낮음.
- **군사**: 태풍 이동속도(~20~40km/h), 기상 전이 확률은 기상청 통계로 고증 가능. 적정.
- **범위**: 적정. 700초 전술 시뮬에선 변화 폭 작음 → **v17 캠페인(72시간)에서 진가**. 전술급에선 효과 미미할 수 있음.

### v12.5 피아식별 오류 (IFF) — 난이도 **중간**(유지)
- **구현**: 공수 ~3~4일. `iff_confidence` 필드 + 교전 보류 로직. 단순.
- **군사**: 매우 현실적(실전 오사 빈발 — 1988 빈센스함, 1994 블랙호크). 식별 지연·오인 확률 고증 가능.
- **범위**: 적정. 다방위·혼잡 시나리오(v11.5)와 시너지.

## v13 — 시각화 & 인터페이스

### v13.1 3D 전장 + 실제 지도 — 난이도 **매우 높음**(유지)
- **구현**: 공수 ~3~5주. VisPy/PyQtGraph 3D + GEBCO/SRTM 로딩 + 좌표 변환. 위험: 대용량 지형 데이터 성능, 렌더러 교체 시 기존 탭 호환.
- **군사**: 무관(표현). 단 레이더 커버리지 '구'는 부정확 — 실제는 수평선·지형 차폐로 비대칭. v9.12 지형음영과 연동해야 현실적.
- **범위**: 매우 큼. **MVP**: 3D는 보류, **2.5D 실제 지도 오버레이(Folium 정적/MapLibre)** 먼저. 3D는 비용 대비 효용 낮음.

### v13.2 C4ISR 네트워크 가시화 — 난이도 **중간**(유지)
- **구현**: 공수 ~1주. NetworkX + 간단 그래프 위젯. 위험 낮음.
- **군사**: 데이터링크 토폴로지(Link-16 메시 vs CEC) 고증 가능. 적정.
- **범위**: 적정. CEC(v10.4) 이미 있으므로 시각화만 추가 — 비용 낮고 교육 가치 높음. **우선순위 상향 권장**.

### v13.3 실시간 다중 KPI 대시보드 — 난이도 **중간→낮음** (하향)
- **구현**: 공수 ~3~4일. 대부분 지표는 기존 stats에서 파생. 신규 계산 적음.
- **군사**: 교환비·비용효율은 표준 군사 OR 지표. 적정.
- **범위**: 적정. v10.7 전술 의사결정과 직결 — **선(先)구현 가치 높음**.

### v13.4 위성 정찰 재방문 주기 — 난이도 **중간**(유지)
- **구현**: 공수 ~1주. 궤도 전파(SGP4)까지 가면 무거움. 위험 중간.
- **군사**: **교정** — 정찰위성 재방문은 단일 위성 기준 12~24h이나, 군집(별자리) 운용 시 수십분~수시간. "OTH 표적지정 위성 의존"은 현대전 정확. 단 지연(이미지→해석→전송) 모델 필수.
- **범위**: 큼. **MVP**: SGP4 궤도전파 보류 → **고정 주기 룩업(pass_times 사전계산)**. v17 캠페인에서 진가, 전술급에선 효용 작음.

## v14 — AI & 자율화

### v14.1 적응형 전술 AI (MCTS) — 난이도 **매우 높음**(유지)
- **구현**: 공수 ~3~4주. MCTS 자체보다 **시뮬을 롤아웃 가능하게 만드는 것**이 난관 — 매 노드마다 SimV7 부분실행 필요 → 느림. 위험 큼.
- **군사**: 적 지휘관이 MCTS로 최적전술을 둔다는 건 "완벽한 적" — 비현실적일 수 있음. 실제 지휘관은 교리·불완전정보 기반. **휴리스틱+불확실성**이 더 현실적.
- **범위**: 욕심. **MVP**: MCTS 전에 **규칙기반 적응 AI**(v10.x ai_tactic 확장) — 손실률 임계마다 전술 전환. MCTS는 그 다음.

### v14.2 ML 기반 즉시 예측 — 난이도 **높음**(유지)
- **구현**: 공수 ~2주. sklearn 회귀(GradientBoosting)면 충분, PyTorch 과함. 위험: 학습데이터 양·대표성.
- **군사**: 무관(메타). 단 예측은 보간일 뿐 — 학습범위 밖 조합은 부정확. "MC 대체"는 과신 금물, **빠른 1차 추정**으로 포지셔닝.
- **범위**: 적정. **v17 캠페인의 핵심 부품** — 이게 있어야 72시간 캠페인이 수초. 우선순위 높음. v11.4(데이터 생성속도) 선행.

### v14.3 함정별 자율 AI — 난이도 **높음**(유지)
- **구현**: 공수 ~2주. `ai_agent` 필드 + CEC두절 시 분기. 위험 중간.
- **군사**: 현대 이지스는 Auto-SM/교전자동화 실재 — 현실적. 자율화 단계(반자동/자동/특수)도 교리적.
- **범위**: 적정. CEC두절(v10.4) 이미 있어 연동 자연스러움.

### v14.4 무인기 군집 Swarm — 난이도 **매우 높음**(유지)
- **구현**: 공수 ~2~3주. Boids는 단순하나 **수백 개체 = 성능 문제** → 공간 분할(quadtree) 필요. _check_hits 부하 급증.
- **군사**: 매우 시의적절(우크라이나전 드론). 단 군집 가격·항속·탑재 현실 반영 필요 — 수백기는 소형(저위협 개별), 수십기는 중형.
- **범위**: 큼. **MVP**: 수백기 보류 → **수십기 + 공간분할**부터. 레이저(v16.2)와 짝.

### v14.5 ISR 드론 군집 — 난이도 **높음**(유지)
- **구현**: 공수 ~1~2주. FriendlyAircraftObj 파생 + 탐지 보정. 위험 낮음.
- **군사**: RQ-4/MQ-9 해상 ISR 현실적. 피격 취약성(저속·고고도)도 반영하면 좋음.
- **범위**: 적정. v14.4보다 선행이 자연스러움(아군 자산이 단순).

## v15 — 전장 도메인 확장

### v15.1 전자전 고도화 + ESM 역탐지 — 난이도 **높음**(유지)
- **구현**: 공수 ~2주. EWLayer 삽입 + ESM 역탐지 이벤트. 위험 중간(탐지 파이프라인 침습).
- **군사**: 매우 현실적 — **레이더 방사 = 위치 노출**은 현대 해전 핵심 딜레마(EMCON). DRFM 기만도 실재. 고증 좋음.
- **범위**: 적정. ARM 방어(v10.x)의 근본 확장. 우선순위 상향 가치.

### v15.2 다전장 연동 (공군·지상) — 난이도 **높음**(유지)
- **구현**: 공수 ~2주. AirDefenseZone 격자 + ROE 체크. 위험 중간.
- **군사**: 합동 ROE·교전구역 분할 현실적. 단 이건 **사실상 작전급 요소** — 전술급 700초에 넣으면 어색.
- **범위**: **재배치 권장** — v18(공군 작전급)의 선행/일부로 흡수가 자연스러움. 전술급에 두면 반쪽.

### v15.3 사이버전 모듈 — 난이도 **높음→중간** (하향)
- **구현**: 공수 ~1주. cfg 플래그 일시 변조 방식이면 단순(CEC강제두절 등 기존 로직 재사용). 위험 낮음.
- **군사**: 추상화 수준 주의 — 사이버 효과를 "확률적 시스템 마비"로 단순화. 정밀 모델은 불가/불필요. 추상 모델로 충분.
- **범위**: 적정. 단 효과가 기존 ECM/CEC두절과 겹침 → **차별화 포인트(지속성·은밀성) 명확히**.

### v15.4 분산 해양작전 DMO — 난이도 **높음**(유지)
- **구현**: 공수 ~2주. 분산 프리셋 + 함정별 독립 교전. 위험 중간.
- **군사**: 미 해군 실제 교리. 단 한국 해군 적용성은 의문(소형함 위주 ≠ DMO 원형). **한국형 변형**으로 재해석 필요.
- **범위**: 적정하나 개념 정교화 필요.
