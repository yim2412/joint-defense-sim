## 1. v21.2 설계 — 합동 화력 지원

### 1-1. 핵심 통찰

협조 타격의 공통 표적으로 쓸 **`EnemyBase`(적 항구·비행장)가 이미 있다** — 출항분담
(`output_share`)·누적 손상(`damage`)·재건(`recovery_h`) 모델까지 완비. v19.4가 만들었고
전략폭격이 이걸 때려 적 출항능력↓ → SLOC 위협↓로 이어지는 경로가 **이미 검증**됐다
(v19.6 통합 MC: +전략폭격 → control +0.13 → 승률 0.90).

→ v21.2는 **새 표적을 만드는 게 아니라, 해군·육군 화력이 이 표적에 닿게 하고 셋을 협조시키는 층**.

### 1-2. 구조 — 기존 층 분리 패턴 답습

`engine_airforce`(v19)·`engine_army`(v20)와 **대칭**. CampaignEngine이 조합 호출.

```
engine_joint.py (신설)
  └ JointFires  — 공유 표적(EnemyBase)에 3군 화력 협조
       ├ 공군 화력 ← AirCampaign strike effort (기존 _apply_strike 위임)
       ├ 해군 화력 ← CampaignEngine 함정 land-attack 재고
       └ 육군 화력 ← ArmyCampaign 지대지 자산 (신설)
```

**표적 소유권 이전 (하위호환 핵심)**
- 현재: `AirCampaign.__init__`이 `build_enemy_bases(cfg)`로 자기 것을 소유.
- 변경: `AirCampaign.__init__(cfg, bases=None)` — **기본인자로 부모 무수정**.
  - `enable_joint_fires` OFF → `bases=None` → 지금처럼 자기 것 생성 → **bit-identical**.
  - ON → CampaignEngine이 목록을 만들어 AirCampaign·JointFires에 **공유 전달**.

### 1-3. 화력 3원

| 군 | 화력 | 상태 | 근거 |
|----|------|------|------|
| 공군 | 전략폭격기(B-1B·B-52) | 기구현 | `_apply_strike` (effort × `_STRIKE_DMG_PER_EFFORT` 0.030) |
| 해군 | Tomahawk Block V (1700km) | DB 기구현 | DDG-51 8 · CG-47 16 · SSN 65 |
| 해군 | 현무-3C (1500km) | **DB 추가** | 세종대왕급 KVLS 48셀 = 현무-3C 32 + 홍상어 16(공개 보도). DB에 이미 홍상어 16 → 정확히 맞물림 |
| 육군 | 현무-2 계열 | **신설** | `engine_army`에 지대지 자산 없음 |

**천무 제외 근거**: 사거리 80km — 적 항구·비행장에 대부분 안 닿는다. _PLANS 문구엔
'천무·현무'가 있으나 **제원상 표적에 도달 못 하는 화력을 넣으면 발동 0인 죽은 기능**이 된다
(부채 청소 교훈: 무대 없는 기능은 만들지 않는다). 육군 화력은 현무 계열로 한정.

### 1-4. ⚠ DB 조사에서 나온 별건 발견 2가지

**① 현무-3C는 현재 죽은 DB 항목**
`FRIENDLY_STRIKE_DB`에 제원이 있고 `_select_sub_strike_wpn`에 전용 코드 경로까지 있는데
**탑재 함정이 0**. 실제 KSS-III(도산안창호급)는 현무-3C용 VLS 6셀 보유인데 DB엔 청상어 12뿐.
→ 코드 경로 有 / 무대 無 = 부채 청소에서 반복 확인한 패턴.

**② 그 죽은 경로는 살리면 안 된다 — 비물리**
`_select_sub_strike_wpn`이 현무-3C를 **적 수상함 공격**에 쓴다. 현무-3C는 지상공격 순항
미사일 = 함정을 치는 무기가 아니다. **SM-3(외기권↔대기권)·SAM(대공↔수중어뢰)와 정확히 같은
계열의 "잘못된 무기가 잘못된 표적"**.

**→ 이번 결정: KSS-III에는 현무-3C를 주지 않는다.** 주는 순간 죽어 있던 비물리 경로가
살아나 잠수함이 지상공격 미사일로 적 함정을 쏘고 골든이 흔들린다 → v21.2와 원인이 섞여
규명이 탁해진다. **수상 이지스함(KDX-III-B1/B2)에만** 추가.

현무-3C는 대함 무기 선택(`_select_strike_wpn`)에 **없으므로 단발 교전에서 불활성 →
골든 무변**이고, 캠페인 합동화력만 소비한다 = v21.2가 필요로 하는 딱 그 형태.

**→ 후속 별건**: `_select_sub_strike_wpn` 현무-3C 대함 사용 비물리 → 별도 seq로 처리
(지금 고치면 v21.2와 원인이 섞임). KSS-III 현무-3C 탑재도 그때 함께.

### 1-5. 협조의 실체 — 두 메커니즘

**① 화력 효과 누적 (overkill 방지)**
현재 `damage += 0.030 × effort` 단순 가산. 이미 반파된 표적에 추가 타격은 낭비 →
증분에 **`(1 - damage)` 체감**을 곱한다(잔존 유효 표적 비율 = 물리적으로 정당한 근거).
협조하면 **잔여 표적으로 화력을 넘겨** 낭비를 줄인다 → 이게 협조의 이득.

**② 시차 vs 동시 (`joint_fire_mode`)**
- **동시(simultaneous)**: 같은 틱 집중 → 적 방공망 분산 → 효과 상승. 단 유인 폭격기와
  순항미사일이 같은 표적·같은 틱 → **협조 없으면 유인 소티 취소**(화력지원 협조수단 위반).
- **시차(sequential)**: 소티 취소 0. 단 적이 틱 사이 재건(`recovery_h`).

**⚠ "아군 오사"를 확률 주사위로 만들지 않는 이유**: 캠페인 층은 SEAD·전략폭격 모두
**결정론**(rng 불사용)으로 통일돼 있고, 작전급 오사 확률값은 **공개 교리에 근거가 없다**.
넣으면 급유기 승수와 같은 **모델 부채**가 된다([[project-db-realism]] · engine_airforce
`_TANKER_MULT_PER` 주석 선례). → **결정론적 소티 취소**로 모델링.

### 1-6. 발현 증거 3종 (커밋 게이트 — 미리 못 박음)

`enable_joint_fires`는 신규 토글이므로 `chk_effect_coverage`가 `audit_effect.py` 프로브 없이는
**커밋 자체를 막는다**. 죽은 채 커밋 금지.

| 증거 | 계획 |
|------|------|
| **② 짝 기능** | **`enable_strategic_strike`가 OFF면 `EnemyBase`가 아예 존재하지 않아 합동화력은 원리상 발동 0.** 부채 청소에서 반복 확인한 패턴이라 **처음부터 짝으로 묶는다** — UI 툴팁·프로브 모두에 명시. `enable_campaign_mode`도 필수(캠페인 전용) |
| **③ 발현 무대** | 한미 연합 편성(토마호크 보유 함정) 또는 세종대왕급(현무-3C) + 전략폭격 ON + 다일 캠페인 horizon |
| **① 발현 델타** | 적 기지 손상·적 출항능력·SLOC 위협·승률 ON/OFF 비교 |

**⚠ 측정 무효 방지**([[patch-queue]] v18.05.09 교훈): 프로브 cfg는 반드시 기존 캠페인 프로브
패턴을 따르고, **표적 수·화력 소스 수를 항상 함께 출력**해 0이면 즉시 측정 실패로 판정.

## 1-7. 구현 후 확정된 사실 (2026-07-16, v21.01.01)

### 발현 증거 3종 — 확보

| 증거 | 실측 |
|------|------|
| **② 짝 기능** | **3중**으로 판명: `enable_joint_fires` + `enable_strategic_strike`(표적 존재) + **`enable_air_campaign`**. 세 번째는 설계 중 발견 — 기지 손상 → 적 출항능력 환산(`enemy_output_factor`)·기지 재건 틱이 **공군 층 소관**이라, 공군이 없으면 표적을 때려도 결과에 반영될 통로가 없다(= 죽은 기능). 하나라도 빠지면 **층을 생성하지 않는다**(프로브로 확증) |
| **③ 발현 무대** | **한미 기동전단 강화 + 최소 방공(제공권 열세) + 입체 포화 (최강)**. 이 편성은 전략폭격기가 없어 OFF면 적 기지를 때릴 수단이 **아예 없다**(출항 1.0=무손상) → 합동 화력의 교리적 의미 그 자체 |
| **① 발현 델타** | 그 무대: **draw→win**·통제 0.667→0.711·적출항 **1.0→0.805**. 강 공군 무대: 기지손상 0.267→0.355·적출항 0.64→0.561. `audit_effect` 등재: 적출항 −0.195·통제 +0.044 |

### 🔴 설계 반례 — (1-damage) 체감은 이중 계상이었다

처음엔 _PLANS 문구("화력 효과 누적 — 첫 타격 손상 → 이후 타격 효과 보정")를 그대로
`(1 - damage)` 표적당 효율 체감으로 구현했다. 결과: **합동 화력이 공군 단독보다 기지를
덜 부수는 역설(0.267 → 0.191)** + 표적넘김 0(= 협조 이득 자체가 발현 안 함).

**원인**: `_BASE_DMG_CAP` 0.80이 **이미** "때려도 분산·예비로 완전 파괴 불가"라는 체감을
표현하고 있었다. 그 위에 또 곱하니 화력이 상한에 **점근만 하고 도달을 못 해** cap 기반
'표적 넘김' 이득이 영영 안 걸렸다. → **v18.05.04(표적 난이도가 탄도에 이중 벌점 → SM-3가
자기 설계 표적에 벌점)와 정확히 같은 유형.**

**결론**: 체감 제거. **협조 이득 = 총화력↑ + 상한 도달 시 잔여 표적 넘김.** 표적당 효율
체감은 `_BASE_DMG_CAP`의 몫이다.

### 협조 모드 — 두 효과는 다른 축

초기 구현은 분산 이득과 오사 대가를 `air_dmg > 0` 하나로 묶었다 → **폭격기 없는 편성
(= 발현 무대!)에서 시차/동시가 bit-identical** = 모드 선택이 죽었다. 분리:
- **분산 이득**(`_SIMULTANEOUS_BONUS`): 화력원 **2개 이상**이면 성립(다축 = 방공망 분산). 폭격기 무관.
- **오사 대가**(`_DECONFLICT_SORTIE_LOSS`): **유인 폭격기가 실제로 뜰 때만** 성립.

실측(폭격기 없는 무대): 시차 기지손상 0.144 → 동시 **0.165**, 협조취소 0(유인기 없으니 정상).

## 5. v21.1 JCS 충돌 경고 — 구현 확정 (2026-07-23, v21.03.01)

### 설계 — 상시 관찰층(토글 없음), 교전 결과 무변경
사용자 합의: **경고만(순수 관찰)** — 자동 조정 없음. 사양("완전 자동최적화는 비현실 →
조정·충돌경고 수준")에 부합하고, 실질 조정(overkill 화력 넘김·CAP 상한)은 이미 하위 층이
수행하므로 그 위에 조정을 얹으면 **v21.2 `(1-damage)` 이중 계상 역설과 같은 유형**이 된다.

`CampaignEngine._jcs_warnings(result)` — 이미 병합된 각 층 지표를 지휘부 관점에서 읽어
명명한 경고 리스트 생성. 토글 없음 → 죽은기능 게이트 대상 아님(v21.4 방식). `enable_*`
신규 없음 → 회귀 bit-identical 보장.

### 🔴 제외한 가지 — 임무 과부하(overtask)
처음엔 4번째 경고로 '가용기<SEAD/CAS 수요'를 넣고 공군에 미충족 계측을 추가했으나,
**배정 시스템이 자기제한적**이라 realistic 무대에서 미발동: ①SEAD는 제압되면 zone이
`ad_active`에서 빠져 수요가 소멸 ②CAS 요청이 희소해 6기(최소 방공)로 늘 충당. 3개 함대
프리셋 × 최소 방공에서 unmet=0 실측 → **죽은 가지로 제외**([[project-dead-feature-prevention]]:
발현 무대 없는 기능은 만들지 않는다). 사양의 두 충돌(표적 겹침=overkill·자산 충돌=deconflict)은
남은 경고로 커버되므로 사양 충족.
